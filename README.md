抓取目标页面：根据用户输入的登录页面 URL，下载其 HTML 内容。

定位登录表单：自动识别包含密码输入框的表单，并记录所有输入字段的名称。

下载图片资源：将页面中的所有图片下载到本地 images 文件夹，并更新 HTML 中的引用路径。

移除 JavaScript：彻底删除所有 <script> 标签（包括外链和内联代码）以及元素上的 on* 事件属性，防止前端逻辑干扰表单提交。

修改表单指向：将表单的 action 改为本地的 save.php，method 设为 post。

生成后端脚本：创建 save.php，接收 username 和 password 字段，将数据追加写入 data.txt，并重定向回用户输入的原始 URL。

输出文件：在脚本所在目录的以目标URL命名的文件夹（防止输出内容都保存到同一文件夹从而导致文件名冲突和内容混淆）
        生成 index.html（钓鱼页面）、save.php（后端处理）和 data.txt（日志文件），部署到支持 PHP 的服务器即可使用。
